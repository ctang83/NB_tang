——————————————————导入数据—————————————————

1.需要确认需要的表空间 1.1创建表空间则不需要remap_tablespace 1.2不创建表空间，可以使用REMAP_TABLESPACE=ehr:users,tbs_phis:users这种语法把ehr和tbs_phis表空间中对象映射到users 1.3确认表空间可以在导入过程中查看报错，报错日志显示有缺少XXX表空间的情况下，按一次CTRL+C，再输入KILL_JOB，再YES，退出报错的导出，删除导入的用户，重新导入 1.4常见表空间（不全）SPS_PHIS、TBS_LOG、TBS_PHIS、PHIS_HIS

2.remap_schema 数据库已经存在的该同名用户报错，可以使用此参数重命名新导入的用户,remap_schema=phis_jys:phis_jiangyou把phis_jys重命名为phis_jiangyou *用户名在导出log文件中，log文件在导出数据中附带。

3.创建directory 3.1确认dmp文件所在位置，例如：dmp位于D:，必须直接能读取到dmp的路径； 3.2用sys/system执行 create or replace directory full as ‘D:’; create or replace directory full as ‘D:’;

打开sqlplus，执行： system/XNHsc2018 create or replace directory full as ‘D:’; *一定要加分号！！！！

3.导入代码 cmd执行 3.1剑阁示例 impdp system/XNHsc2018@localhost/oracledb dumpfile=jg180915_%U.dmplogfile=jg180915_imp.log directory=full remap_tablespace=SPS_PHIS:users,TBS_LOG:users,TBS_PHIS:users,PHIS_HIS:users parallel=6 cluster=n *标黄部分：需要根据导出日志进行修改

3.2东坡示例 impdp system/XNHsc2018@localhost/oracledb dumpfile=dp180914.dmp logfile=dp180914_imp.log directory=full remap_tablespace=SPS_PHIS:users,TBS_LOG:users,TBS_PHIS:users,PHIS_HIS:users parallel=6 cluster=n 3.3dazhu impdp system/XNHsc2018@localhost/oracledb dumpfile=dazhu180920_%U.dmp logfile=dazhu180920_imp.log directory=full remap_tablespace=SPS_PHIS:users,TBS_LOG:users,PHIS_DAZHU:users,PHIS_HIS:users parallel=6 cluster=n

XNHsc2018为system密码； oraclebd为正确的实例名； 修改dmp文件名（Linux平台区分大小写）； 修改合适的remap_tablespace参数（如果表空间有改变，需要修改）
